The network that is created is based on proximity (Euclidean distance) between nodes. A node is randomly chosen and connected to the nearest node that it is not already connected to. This process is repeated until the network has the correct number of links to give the specified average node degree.
### Spanish
La red que se crea se basa en proximidad (distancia Euclidiana) entre nodos. Un nodo se elige al azar y se conecta al nodo más cercano al que aún no está conectado. Este proceso se repite hasta que la red tiene el número correcto de enlaces para dar el grado de nodo promedio especificado.

This model maps <color=orange>turtles</color> between <trigger=cartesian-coordinates>Cartesian</trigger> and <trigger=spherical-coordinates>spherical</trigger> 3-dimensional coordinates. To create the appearance of a curved 3D surface, the program generates <color=orange>turtles</color> with random x- and z-coordinates, then computes each <color=orange>turtle's</color> y-coordinate based on x and z, and time . The <trigger=cosine>cosine</trigger> function is used in the formula to produce a curved surface. Varying the y coordinate based on time  produces motion.

### Source
To render the surface in the NetLogo view, it maps from 3D to 2D coordinates as follows:
view x = turtle x + (turtle z) / 2 
view y = turtle y + (turtle z) / 2
In other words, increasing z causes the <color=orange>turtle's</color> apparent position to move both up  and to the right .
### Spanish
Para renderizar la superficie en la vista de NetLogo, se asigna de coordenadas 3D a 2D de la siguiente manera:
ver x = tortuga x + (tortuga z) / 2
ver y = tortuga y + (tortuga z) / 2
En otras palabras, aumentar z provoca la posición aparente de la <color=orange>tortuga</color> para moverse tanto hacia arriba  como hacia la derecha .
